What is this `git rebase` you are talking about?


From git-scm.com:
git-rebase - Reapply commits on top of another base tip

Zuvor erwähnen: Rebasing hat keinen Einfluss auf das "GIT Flow" Konzept.
---
Übung:
man möchte den Stand aus master in den feature branch übernehmen
git merge master
 Zeige zwei Bäume, die dauerhaft bestehen bleiben
reset
clear
git rebase master
 Zeige einen Baum
---
Rebasing ist die bessere Alternative zu `merge`, um Code nachzuziehen denn es hält den Commitbaum sauberer.
Informationen über Branches kann man den Commitmessages entnehmen, falls überhaupt relevant
Rebasing interactive
  • Remove C3
  • Move C4 before C2
Showtime in Shell and IDE
  • Show pick, squash, reword, reorder
  • Those should be the most used actions
